# 开发流程

1、注册小程序：获取appid、安装开发者工具、绑定开发者、体验者等 2、代码开发：框架选择、ui框架选择、代码结构、公用文件抽取等 3、代码上传：体验版、审核版、线上版 4、代码发布：填写审核信息、审核通过或不通过

审核不通过原因：

1、测试账号不符合规则，部分功能无法完整体验，请在下一次提交审核时提供测试账号；

解决办法：通知消息下的反馈表单里填写测试账号，代码再提交审核

# 登录流程

说明： 调用 wx.login() 获取 临时登录凭证code ，并回传到开发者服务器。 调用 auth.code2Session 接口，换取 用户唯一标识 OpenID 和 会话密钥 session_key。 之后开发者服务器可以根据用户标识来生成自定义登录态，用于后续业务逻辑中前后端交互时识别用户身份。

注意：

1.会话密钥 session_key 是对用户数据进行 加密签名 的密钥。为了应用自身的数据安全，开发者服务器不应该把会话密钥下发到小程序，也不应该对外提供这个密钥。 2.临时登录凭证 code 只能使用一次
